# 🎯 聊天卡片示例数据使用指南

## 📋 概述

我们已经创建了一个包含所有类型卡片的示例聊天会话，方便你在聊天列表中查看和测试各种卡片样式。

## 🎨 包含的卡片类型

示例会话包含 **7 种不同类型的卡片**：

| 卡片类型 | 组件名 | 描述 | 示例内容 |
|---------|--------|------|---------|
| 📅 **EVENT** | `EventCard` | 日程确认卡片 | 项目周会 |
| ✅ **TASK** | `TaskCard` | AI 任务拆解卡片 | 编写 API 文档（4 个子任务） |
| 🔔 **ALERT** | `AlertCard` | 提醒卡片 | 喝水提醒 |
| 📊 **SCHEDULE** | `ScheduleCard` | 时间轴卡片 | 今日时间轴 |
| 📈 **GRAPH** | `GraphCard` | 统计图表卡片 | 专注时长统计 |
| 📝 **EVENT_LIST** | `EventListCard` | 事件列表卡片 | 今日待办事项（3 个事件） |
| 💬 **SESSION** | `SessionCard` | 会话列表项 | 侧边栏使用 |

## 🚀 快速开始

### 🎛️ 统一开关控制（推荐）

所有调试功能的开关都集中在 **`lib/app/constants/debug_config.dart`** 文件中。

**步骤：**

1. 打开文件 `lib/app/constants/debug_config.dart`

2. 找到以下配置项：
   ```dart
   class DebugConfig {
     const DebugConfig._();

     /// 是否启用调试模式
     static const bool isDebugMode = true;

     // ==================== 示例数据 ====================

     /// 是否启动时自动创建示例聊天会话
     static const bool createSampleChatSession = true;  // 👈 修改这里

     /// 是否自动删除旧的示例会话
     static const bool autoRemoveOldSampleSessions = false;
   }
   ```

3. 修改 `createSampleChatSession` 的值为 `true` 或 `false`：
   - `true` - 启用示例数据（开发/测试环境）
   - `false` - 禁用示例数据（生产环境）

4. 重新运行应用即可

**优势：**
- ✅ 所有调试开关集中管理，一目了然
- ✅ 生产/开发环境切换方便
- ✅ 避免遗漏注释代码
- ✅ 可扩展其他调试功能

### 方法二：在代码中手动调用

在任何需要的地方导入并调用：

```dart
import 'package:planly_ai/app/data/sample_chat_session.dart';
import 'package:planly_ai/app/constants/debug_config.dart';

// 创建示例数据
await createSampleChatSessionWithAllCards();

// 删除示例数据（可选）
await removeSampleChatSessions();
```

### 方法三：添加测试按钮

在设置页面或调试页面添加一个按钮：

```dart
ElevatedButton(
  onPressed: () async {
    await createSampleChatSessionWithAllCards();
    if (mounted) {
      ScaffoldMessenger.of(context).showSnackBar(
        const SnackBar(content: Text('✅ 示例聊天会话创建成功！')),
      );
    }
  },
  child: const Text('🎨 创建卡片展示示例'),
)
```

## 📱 查看效果

启用示例数据后：

1. 启动应用
2. 进入 **聊天机器人** 页面
3. 在聊天列表中找到 **"卡片展示示例"** 会话
4. 点击进入即可看到所有类型的卡片

## 🗑️ 清除示例数据

如需删除示例会话，可以调用：

```dart
import 'package:planly_ai/app/data/sample_chat_session.dart';

await removeSampleChatSessions();
```

这会删除所有 `sessionId` 以 `sample_session_` 开头的会话。

### 数据类型定义

所有卡片数据都存储在 `ChatMessage` 的 `cardContent` 字段中（JSON 格式）：

```dart
enum MessageType {
  text,
  image,
  file,
  voice,
  scheduleConfirmation,
  focusDuration,
  scheduleBreakdown,
  timelineSchedule,
  cardEvent,        // EVENT 卡片
  cardTask,         // TASK 卡片
  cardAlert,        // ALERT 卡片
  cardGraph,        // GRAPH 卡片
  cardSchedule,     // SCHEDULE 卡片
  cardEventList,    // EVENT_LIST 卡片
}
```

### 渲染逻辑

在 `chat_bubble.dart` 中根据 `MessageType` 渲染对应的卡片：

```dart
switch (message.type) {
  case MessageType.cardEvent:
    return _buildCardWithAvatar(context, colorScheme, _buildEventCard());
  case MessageType.cardTask:
    return _buildCardWithAvatar(context, colorScheme, _buildTaskCard());
  // ... 其他卡片类型
}
```

## ⚠️ 注意事项

1. **仅用于开发测试**：示例数据仅用于开发和测试目的，不建议在生产环境使用
2. **自动清理**：示例会话的 `sessionId` 以 `sample_session_` 开头，便于识别和清理
3. **重复创建**：每次调用 `createSampleChatSessionWithAllCards()` 都会创建新的会话，不会覆盖已有会话
4. **调试模式**：建议在调试模式下使用，生产环境请禁用

## 🆘 常见问题

### Q: 如何删除所有示例数据？
A: 调用 `removeSampleChatSessions()` 函数即可删除所有示例会话。

### Q: 示例数据会影响真实数据吗？
A: 不会。示例数据是独立的会话，与真实数据分开存储。

### Q: 可以修改示例数据的内容吗？
A: 可以。直接编辑 `sample_chat_session.dart` 文件中的 JSON 数据即可。

### Q: 为什么启动时没有看到示例会话？
A: 请确保已经取消了 `main.dart` 中 `createSampleChatSessionWithAllCards()` 的注释。
